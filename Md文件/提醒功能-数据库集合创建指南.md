# 提醒功能 - 云数据库集合创建指南

## 概述

提醒功能需要以下三个云数据库集合，请在微信开发者工具的**云开发控制台**中手动创建。

---

## 1. user_settings - 用户设置集合

### 用途
存储用户的提醒设置、偏好配置等。

### 权限设置
```json
{
  "read": "doc._openid == auth.uid",
  "write": "doc._openid == auth.uid"
}
```

### 字段结构

| 字段 | 类型 | 说明 | 示例 |
|-----|------|------|------|
| _id | string | 自动生成 | - |
| _openid | string | 用户OpenID（自动生成） | - |
| reminder | object | 提醒设置对象 | 见下方 |
| createdAt | date | 创建时间 | 2024-01-01 10:00:00 |
| updatedAt | date | 更新时间 | 2024-01-01 10:00:00 |

**reminder 对象结构：**
```json
{
  "enabled": true,
  "time": "09:00",
  "updatedAt": "2024-01-01T10:00:00.000Z"
}
```

### 创建步骤
1. 打开微信开发者工具
2. 点击左侧「云开发」→「数据库」
3. 点击「+」按钮，输入集合名称 `user_settings`
4. 创建后，点击「权限设置」
5. 选择「自定义」，粘贴上方权限JSON
6. 点击「确定」保存

---

## 2. todos - 待办事项集合

### 用途
存储所有用户的待办事项，供定时提醒云函数扫描。

### 权限设置
```json
{
  "read": "doc._openid == auth.uid",
  "write": "doc._openid == auth.uid"
}
```

### 字段结构

| 字段 | 类型 | 说明 | 示例 |
|-----|------|------|------|
| _id | string | 自动生成 | - |
| _openid | string | 用户OpenID（自动生成） | - |
| id | string | 本地待办ID | "1704067200000_abc123" |
| text | string | 待办内容 | "买菜" |
| priority | string | 优先级 | "urgent-important" |
| done | boolean | 是否完成 | false |
| date | string | 日期（YYYY-MM-DD） | "2024-01-01" |
| createdAt | date | 创建时间 | 2024-01-01 10:00:00 |
| updatedAt | date | 更新时间 | 2024-01-01 10:00:00 |

### 索引建议
| 索引字段 | 排序方式 | 用途 |
|----------|------------|------|
| _openid | 升序 | 查询用户待办 |
| date | 升序 | 按日期查询 |
| done | 升序 | 筛选未完成 |

### 创建步骤
1. 在云开发控制台，点击「+」创建集合 `todos`
2. 设置权限（同上）
3. 点击「索引管理」→「添加索引」
4. 添加上述索引

---

## 3. push_logs - 推送日志集合

### 用途
记录每次提醒推送的日志，用于排查问题和统计分析。

### 权限设置
```json
{
  "read": "doc._openid == auth.uid",
  "write": "doc._openid == auth.uid"
}
```

### 字段结构

| 字段 | 类型 | 说明 | 示例 |
|-----|------|------|------|
| _id | string | 自动生成 | - |
| _openid | string | 用户OpenID（自动生成） | - |
| todoCount | number | 未完成待办数量 | 3 |
| pushTime | date | 推送时间 | 2024-01-01 09:00:00 |
| status | string | 推送状态 | "pending" / "sent" / "failed" |
| errorMessage | string | 错误信息（可选） | "用户未授权" |
| createdAt | date | 创建时间 | 2024-01-01 09:00:00 |

### 创建步骤
1. 在云开发控制台，点击「+」创建集合 `push_logs`
2. 设置权限（同上）

---

## 快速创建（使用云函数初始化）

您也可以创建一个初始化云函数，运行一次来创建集合和索引。

### initCollections 云函数

```javascript
// cloudfunctions/initCollections/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  try {
    // 创建集合（如果不存在）
    const collections = ['user_settings', 'todos', 'push_logs']
    
    for (const name of collections) {
      try {
        await db.createCollection(name)
        console.log(`集合 ${name} 创建成功`)
      } catch (e) {
        // 集合可能已存在
        console.log(`集合 ${name} 已存在或创建失败`, e.message)
      }
    }

    return { success: true, message: '集合初始化完成' }
  } catch (err) {
    return { success: false, error: err.message }
  }
}
```

### 使用步骤
1. 创建 `initCollections` 云函数
2. 上传并部署
3. 在云开发控制台 → 「云函数」→ 找到 `initCollections`
4. 点击「运行」，执行一次
5. 检查数据库，确认集合已创建
6. **删除此云函数**（安全考虑）

---

## 验证集合创建成功

在微信开发者工具中，进入「云开发」→「数据库」：

1. 确认 `user_settings`、`todos`、`push_logs` 三个集合存在
2. 点击每个集合，确认权限设置正确
3. 手动添加一条测试数据，验证读写正常

---

## 注意事项

1. **权限设置非常重要**：错误的权限会导致用户无法读写自己的数据
2. **集合名称必须完全匹配**：`user_settings`、`todos`、`push_logs`
3. **首次使用前先测试**：用两个微信账号测试完整的提醒流程
4. **定期清理日志**：`push_logs` 集合会持续增长，建议定期删除30天前的日志
