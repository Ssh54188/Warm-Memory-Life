## 一、产品概述

### 1.1 产品定位

一款手账风格的周计划小程序，融合待办、习惯打卡、记事本、日历、四象限五大核心模块，帮助用户以「周」为单位组织生活。支持邮箱 / 手机号 / 微信一键登录，数据云端同步。内置艾森豪威尔四象限矩阵，按重要性与紧急性分类管理任务，支持拖拽跨象限迁移。

### 1.2 目标用户

- 学生群体：管理课业任务和学习计划
- 职场人群：跟踪工作进度和日常习惯
- 手账爱好者：偏好温暖视觉风格的个人管理工具

### 1.3 核心价值

- 以周为节奏，不强求日粒度压力
- 概览仪表盘快速掌握本周状态
- 数据云端同步，手机随时查看编辑

## 三、登录系统

### 3.1 登录方式

| 方式 | 说明 |
|------|------|
| 微信一键登录 | `wx.login()` 获取 code → 后端换取 openid / unionid，无需额外输入 |
| 手机号登录 | `button open-type="getPhoneNumber"` 获取加密手机号 → 后端解密 |
| 邮箱登录 | 邮箱 + 密码，后端校验，适合非微信场景的 Web 端同步 |

### 3.2 登录流程

1. 首次启动展示登录页，三种方式 Tab 切换
2. 微信登录：点击授权按钮 → `wx.getUserProfile()` 获取昵称头像 → 自动登录
3. 手机号登录：点击按钮获取微信绑定手机号 → 后端返回 token
4. 邮箱登录：输入邮箱密码 → 后端校验 → 返回 token
5. 所有方式登录成功后，将 token 存入 `wx.getStorageSync('token')`
6. 支持「跳过登录」进入本地模式，数据仅存于 `wx.getStorageSync`，不调用后端

### 3.3 登录态管理

- `App.onLaunch()` 检查本地 token，有效则自动恢复登录
- 无 token 则跳转登录页
- 个人中心提供退出登录按钮，退出后清空 token，数据保留在本地

## 九、四象限（优先级管理）

### 9.1 设计理念

基于艾森豪威尔矩阵（Eisenhower Matrix），将任务按「重要性」与「紧急性」划分到四个象限，帮助用户聚焦真正重要的事。四象限数据与每日待办完全独立，互不干扰。

### 9.2 四象限布局

```
┌──────────────────────────────┐
│                 紧急 →       │
│  ┌────────────┬────────────┐ │
│  │ ① Q1       │ ② Q2       │ │
│  │ 重要且紧急  │ 重要不紧急  │ │
│  │ [输入框]   │ [输入框]   │ │
重│  ☐ 紧急任务1 │ ☐ 学习计划  │ │
要│  ☐ 突发事务  │ ☐ 健身安排  │ │
│  │ [+添加]    │ [+添加]    │ │
↓ ├────────────┼────────────┤ │
│  │ ③ Q3       │ ④ Q4       │ │
│  │ 不重要紧急  │ 不重要不紧急│ │
│  │ [输入框]   │ [输入框]   │ │
│  │ ☐ 临时会议  │ ☐ 刷视频   │ │
│  │ [+添加]    │ [+添加]    │ │
│  └────────────┴────────────┘ │
│                              │
│  Q1: 2项  Q2: 3项  Q3: 1项  Q4: 3项  │ ← 统计行
└──────────────────────────────┘
```

### 9.3 四个象限定义

| 象限 | 标签 | 颜色 | 说明 |
|------|------|------|------|
| Q1 | 重要且紧急 | `#e74c3c`（红） | 危机、截止日期驱动的紧急事项，必须立刻做 |
| Q2 | 重要不紧急 | `#27ae60`（绿） | 长期规划、能力提升，应投入最多时间 |
| Q3 | 不重要但紧急 | `#f39c12`（橙） | 可委托他人的事务，尽量减少 |
| Q4 | 不重要不紧急 | `#95a5a6`（灰） | 消磨时间的活动，应避免或限制 |

### 9.4 核心功能

**任务管理**
- 每象限顶部配输入框，输入任务文本后回车或点击添加按钮新增
- 每条任务：复选框 + 任务文本 + 删除图标
- 勾选后任务文本添加删除线并淡化，表示已完成
- 双击任务文本进入编辑模式，修改后失焦自动保存
- 删除按钮（图标 × 或 🗑）移除此条任务

**拖拽跨象限迁移**
- 长按任务项触发拖拽状态（视觉反馈：轻微放大 + 半透明）
- 拖动到目标象限区域松手，任务从来源象限移除并插入目标象限
- 小程序实现方案：使用 `movable-view` 组件 + `bindchange` 事件监听位置，或 touch 事件三件套（`bindtouchstart` / `bindtouchmove` / `bindtouchend`）计算落点
- 拖拽过程中目标象限高亮提示可放置区域

**统计行**
- 四象限底部实时显示每个象限当前任务总数

### 9.5 入口位置

- 概览页新增「四象限」卡片入口，点击跳转 `/pages/quadrant/quadrant`
- 「我的」页面新增「四象限」菜单项

### 9.6 数据独立性

四象限任务与每日待办（第五章）使用完全独立的存储键，两者互不影响：

```json
{
  "_quadrants": {
    "urgent-important": [
      { "id": "uuid-q1-1", "text": "紧急任务A", "done": false },
      { "id": "uuid-q1-2", "text": "突发事务处理", "done": true }
    ],
    "not-urgent-important": [
      { "id": "uuid-q2-1", "text": "学习Rust", "done": false }
    ],
    "urgent-not-important": [
      { "id": "uuid-q3-1", "text": "临时会议", "done": false }
    ],
    "not-urgent-not-important": [
      { "id": "uuid-q4-1", "text": "刷短视频", "done": true }
    ]
  }
}
```

| 字段 | 说明 |
|------|------|
| `_quadrants` | 四象限根键，与待办 `date_YYYY-MM-DD` 并列 |
| `urgent-important` | Q1：重要且紧急 |
| `not-urgent-important` | Q2：重要不紧急 |
| `urgent-not-important` | Q3：不重要但紧急 |
| `not-urgent-not-important` | Q4：不重要不紧急 |

### 9.7 与每日待办的区别

| 维度 | 每日待办 | 四象限 |
|------|----------|--------|
| 时间粒度 | 按天分组 | 无时间维度 |
| 分类方式 | 日期 | 重要性 × 紧急性 |
| 完成标记 | 勾选后移入已完成区 | 勾选后保留原位（删除线） |
| 移动能力 | 无跨天移动 | 支持拖拽跨象限迁移 |
| 独立性 | 数据键 `date_YYYY-MM-DD` | 数据键 `_quadrants` |

## 十一、云端同步

### 11.1 同步策略

- 登录用户自动启用云端同步
- 每次数据变更（添加待办、勾选、编辑笔记）后延迟 2 秒上传差异
- 登录时拉取云端最新数据，与本地合并（云端优先）
- 同步按钮手动触发全量同步
- 冲突策略：时间戳比较，后修改的为准（last-write-wins）

### 11.2 云开发方案

- 使用微信云开发（CloudBase）
- 数据存储：云数据库，每个用户一个文档，userId 为 `_id`
- 用户系统：云函数 + `wx.cloud.callFunction()`
- 邮箱登录需自建云函数校验

## 十二、技术方案

### 12.1 前端

- 框架：原生微信小程序（或 Taro / uni-app 跨端方案）
- UI：自定义组件 + WXSS，风格延续手账暖色调
- 状态管理：Page 级 data + `wx.getStorageSync` 持久化
- Canvas：概览进度环形图使用 Canvas 2D API

### 12.2 后端（云开发）

- 用户认证云函数：处理微信登录、手机号解密、邮箱校验
- 数据同步云函数：读写云数据库
- 数据库集合：`users`、`planner_data`

### 12.3 本地存储键设计

| 键名 | 内容 |
|------|------|
| `weekly-planner-data` | 全部业务数据 JSON |
| `weekly-planner-user` | 用户登录状态 |
| `token` | 登录凭证 |
