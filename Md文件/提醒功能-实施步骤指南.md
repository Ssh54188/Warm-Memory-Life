# 提醒功能 - 实施步骤指南

## 概述

本文档提供从零开始实施提醒功能的完整10步指南。

---

## 第1步：开通云开发

### 目标
为小程序开通云开发环境，获取环境ID。

### 操作步骤
1. 打开 **微信开发者工具**
2. 点击顶部菜单 「云开发」
3. 如未开通，按提示开通（需微信扫码确认）
4. 开通后，复制 **环境ID**（格式如 `cloud1-xxxxxx`）

### 验证
- 云开发控制台能正常打开
- 能看到「数据库」「云函数」「存储」等标签

---

## 第2步：初始化云环境

### 目标
在 `app.js` 中正确初始化云开发环境。

### 操作步骤
1. 打开 `app.js`
2. 确认已有以下代码（如没有则添加）：

```javascript
// 必须在 App() 注册之前初始化云环境
if (wx.cloud) {
  wx.cloud.init({
    env: '您的环境ID'  // 替换为第1步获取的环境ID
  })
}
```

3. 替换 `'cloud1-d4go8n0ph6464342d'` 为您的实际环境ID

### 验证
- 编译项目，控制台无 `wx.cloud is not defined` 错误
- 云函数调用能正常发起

---

## 第3步：申请订阅消息模板

### 目标
在微信公众平台申请订阅消息模板，用于发送提醒。

### 操作步骤
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「功能」→「订阅消息」
3. 点击「选用」→「申请模板」
4. 搜索关键词：`待办`、`提醒`、`任务`
5. 选择合适的模板（或申请自定义模板）

### 模板示例（参考）

**模板标题**：待办提醒通知

**关键词**（需按顺序）：
1. 待办标题
2. 提醒时间
3. 备注

**模板内容示例**：
```
您有 {{keyword1.DATA}} 待办未完成
提醒时间：{{keyword2.DATA}}
{{keyword3.DATA}}
```

### 获取模板ID
申请通过后，复制 **模板ID**（格式如 `xxxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`）

### 配置到项目
1. 打开 `utils/subscribeMessage.js`
2. 找到 `TEMPLATE_IDS` 常量
3. 替换为实际模板ID：

```javascript
const TEMPLATE_IDS = [
  '您的模板ID'  // 替换为实际模板ID
]
```

### 验证
- 模板状态为「已选用」
- `utils/subscribeMessage.js` 中的 `TEMPLATE_IDS` 已配置

## 第4步：创建云数据库集合

### 目标
创建 `user_settings`、`todos`、`push_logs` 三个集合，并配置权限。

### 操作步骤
详见 [数据库集合创建指南](./提醒功能-数据库集合创建指南.md)

快速步骤：
1. 打开云开发控制台 → 「数据库」
2. 创建集合 `user_settings`
3. 创建集合 `todos`
4. 创建集合 `push_logs`
5. 为每个集合设置权限：

```json
{
  "read": "doc._openid == auth.uid",
  "write": "doc._openid == auth.uid"
}
```

### 验证
- 三个集合在控制台可见
- 权限设置正确（点击集合 → 「权限设置」）

## 第8步：配置定时触发器

### 目标
为 `scheduler` 云函数配置定时触发器，实现每天定时扫描。

### 操作步骤
1. 打开云开发控制台 → 「云函数」
2. 找到 `scheduler` 函数，点击函数名
3. 点击「配置」→「定时触发器」
4. 点击「添加触发器」

### 触发器配置示例

**每天上午9点执行**：
- 名称：`daily-reminder-morning`
- 触发周期：自定义
- Cron表达式：`0 9 * * *`
- 启用状态：启用

**每天晚上8点执行**：
- 名称：`daily-reminder-evening`
- Cron表达式：`0 20 * * *`

### Cron表达式说明
```
* * * * *
│ │ │ │ │ │
│ │ │ │ │ └── 星期（0-6，0=周日）
│ │ │ │ └──── 月份（1-12）
│ │ │ └──────── 日期（1-31）
│ │ └────────── 小时（0-23）
│ └───────────── 分钟（0-59）
└──────────────── 秒（0-59，可选）
```

### 验证
- 触发器状态为「启用」
- 查看「触发历史」确认执行记录

## 附录：故障排查

### 问题1：云函数调用失败

**现象**：控制台报错 `cloud.callFunction:fail`

**排查步骤**：
1. 检查云函数是否已部署
2. 检查云环境ID是否正确
3. 检查云函数名是否拼写正确
4. 查看云函数运行日志

**解决**：
```javascript
// 在调用前打印环境ID
console.log('云环境ID:', wx.cloud.ENV)
```

---

### 问题2：订阅授权失败

**现象**：点击「去授权」无反应或报错

**排查步骤**：
1. 检查 `TEMPLATE_IDS` 是否配置正确
2. 检查模板是否已「选用」
3. 检查小程序是否已发布（订阅消息需要线上版本）

**解决**：
- 在微信开发者工具中，使用「真机调试」测试授权
- 确保模板ID格式正确（含横杠）

---

### 问题3：定时触发器未执行

**现象**：到达设定时间后，`push_logs` 无新记录

**排查步骤**：
1. 检查触发器状态是否为「启用」
2. 查看「触发历史」是否有执行记录
3. 检查 Cron 表达式是否正确
4. 检查云函数代码是否有语法错误

**解决**：
- 手动点击「运行」测试云函数
- 检查云函数运行日志

---

### 问题4：待办未同步到云端

**现象**：本地添加待办后，云端 `todos` 集合无数据

**排查步骤**：
1. 检查 `wx.cloud` 是否初始化
2. 检查云函数 `syncTodos` 是否已部署
3. 查看控制台是否有报错
4. 检查集合权限是否正确

**解决**：
- 在 `todo.js` 的 `_syncTodosToCloud` 方法中添加更多日志
- 手动调用云函数测试
