# 暖记生活小程序需求文档

## 1. 产品概述

**产品名称**：暖记生活  
**产品定位**：手账风格的个人周计划与习惯管理小程序  
**目标用户**：需要规划每周任务、培养习惯、记录日常的用户  
**核心价值**：提供纸质手账般的温暖体验，结合数字化便捷性，帮助用户高效管理一周事务

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 待办事项管理（P0）
- **按天分类**：周一至周日独立待办列表
- **基础操作**：添加、完成、删除待办事项
- **优先级标签**：高/中/低优先级色彩标记（P1）
- **历史记录**：查看已完成事项，支持按天筛选
- **数据统计**：总待办数、完成数、完成率

#### 2.1.2 习惯打卡系统（P0）
- **习惯列表**：自定义习惯名称，支持快捷预设
- **周视图表格**：习惯×星期（7天）的打卡网格
- **打卡操作**：点击切换打卡状态，即时视觉反馈
- **统计面板**：今日完成数、本周完成数、完成率百分比
- **习惯管理**：添加、删除习惯，防重复校验

#### 2.1.3 记事本功能（P0）
- **按天独立**：每天有独立的记事本区域
- **手账风格**：横线纸张纹理，手写字体输入
- **自动保存**：输入延迟自动保存到本地
- **字数统计**：实时显示行数、字数
- **历史查看**：可查看和编辑任意天的笔记
- **独立导出**：每篇笔记可单独导出为 .txt 文件

#### 2.1.4 日历视图（P0）
- **月历网格**：手账风格月视图，按周排列日期
- **笔记标记**：有笔记的日期显示圆点标记和内容预览
- **日期跳转**：点击日期自动跳转到对应笔记编辑页
- **月份切换**：支持上下月翻页和回到今天
- **日期笔记独立存储**：每个日期拥有独立的笔记空间

#### 2.1.5 数据管理（P0）
- **数据导出**：生成JSON备份，支持复制和下载
- **数据导入**：粘贴JSON数据恢复备份，跨设备迁移
- **数据统计**：总待办数、习惯数、完成率等概览
- **数据清空**：带确认提示的完整重置功能

### 2.2 扩展功能（优先级排序）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P1 | 周视图概览 | 一页展示整周待办和打卡总览 |
| P1 | 待办优先级标签 | 高/中/低优先级色彩标记 |
| P2 | 番茄钟计时器 | 内置专注计时，关联待办事项 |
| P2 | 数据可视化 | 打卡趋势图、完成率周报 |
| P3 | PWA支持 | 离线访问、桌面快捷方式 |
| P3 | 多周管理 | 支持查看往周数据 |

## 3. 技术架构

### 3.1 技术栈选择
- **前端框架**：微信小程序原生开发
- **UI组件库**：自定义手账风格组件
- **数据存储**：微信小程序本地存储（wx.setStorage）
- **云服务**：可选微信云开发（数据同步、多端同步）

### 3.2 数据结构设计

```javascript
// 核心数据结构
const data = {
  // 习惯列表
  _habits: ["运动30分钟", "阅读30分钟", "冥想10分钟"],
  
  // 每周数据（周一至周日）
  mon: {
    todos: [
      { id: 123456, text: "完成周报", done: true, priority: "high" }
    ],
    habitChecks: {
      "运动30分钟": true,
      "阅读30分钟": false
    },
    note: "今天天气很好..."
  },
  // ... 其他星期数据
  tue: { todos: [], habitChecks: {}, note: "" },
  wed: { todos: [], habitChecks: {}, note: "" },
  thu: { todos: [], habitChecks: {}, note: "" },
  fri: { todos: [], habitChecks: {}, note: "" },
  sat: { todos: [], habitChecks: {}, note: "" },
  sun: { todos: [], habitChecks: {}, note: "" }
}
```

### 3.3 存储策略
- **本地存储**：使用 `wx.setStorageSync` 存储用户数据
- **备份机制**：JSON导出/导入实现数据迁移
- **容量限制**：小程序本地存储上限10MB，足够使用

## 4. 界面设计规范

### 4.1 视觉风格
- **色彩体系**：
  - 纸张底色：`#faf6f0`（米白纸张）
  - 主色调：`#c4956a`（木质/手账棕）
  - 辅助色：`#f8e8e0`（粉）、`#d4e9d6`（绿）、`#dce8f5`（蓝）
  - 文字色：`#5c4b3c`（深棕）、`#8b7a6b`（浅棕）
- **字体**：手写字体（如"Ma Shan Zheng"或系统手写体）
- **纹理**：纸张纹理、横线纹理、轻微阴影

### 4.2 布局结构
1. **顶部导航**：星期切换（周一至周日）
2. **标签页切换**：待办事项、习惯打卡、记事本、数据管理
3. **内容区域**：各功能模块主要内容
4. **底部操作**：输入框、添加按钮等

### 4.3 交互设计
- **点击反馈**：轻微缩放、颜色变化
- **动画效果**：打卡按钮弹出动画、切换淡入淡出
- **加载状态**：数据加载时的骨架屏
- **错误提示**：友好的错误提示和恢复建议

## 5. 页面规划

### 5.1 主要页面
1. **首页**：周视图概览（待办+打卡总览）
2. **待办页**：按天分类的待办事项管理
3. **习惯页**：习惯打卡表格和统计
4. **记事本页**：每日笔记记录
5. **数据页**：数据管理、备份恢复

### 5.2 页面流程
```
启动 → 首页（周概览）
  ├─ 待办页（添加/管理待办）
  ├─ 习惯页（打卡/管理习惯）
  ├─ 记事本页（记录笔记）
  └─ 数据页（备份/恢复/统计）
```
